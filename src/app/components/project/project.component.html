Información del proyecto Web:


Resumen:

Este proyecto es una versión a pequeña escala de una aplicación web Full-Stack. El front end está desarrollado utilizando Angular 17, mientras que el back end se ha implementado en Node.js. Además, la aplicación cuenta con una base de datos MySQL, que gestiona y almacena la información necesaria para su correcto funcionamiento.


Motivaciones:

El principal objetivo de este proyecto ha sido ampliar mis conocimientos en el desarrollo de aplicaciones web. Comenzó como un ejercicio para aprender y mejorar mis habilidades en Angular 17, pero tras un tiempo de estudio y práctica, decidí expandir su alcance. Incorporé un componente de back end utilizando Node.js y una base de datos MySQL, lo que me permitió estudiar y profundizar en los métodos de comunicación dentro de una aplicación web Full-Stack.

Primer Incremento:


Configuracion y estructura:

En este primer incremento, realicé la configuración de la parte front-end de mi aplicación web. Posteriormente, me familiaricé con la arquitectura de Angular 17 y sus modificaciones en comparación con versiones anteriores.

Es importante destacar que Angular 17 permite el uso de componentes independientes (standalone components), lo que significa que el enrutamiento se puede gestionar directamente en el archivo app.routes.ts, sin necesidad de definir módulos en la misma forma que se hacía en versiones anteriores.


Una vez que asimilé los conceptos de Angular, decidí estructurar de manera eficiente el directorio app. Para ello, creé un directorio llamado components, donde almaceno todos los componentes generados. Además, establecí un segundo directorio llamado src, que a su vez contiene las carpetas directives, guards, layout, iframe-video y services. En cada una de estas carpetas, se encuentran las directivas, guards, servicios, etc., organizados y separados según su uso específico y características. Para facilitar las importaciones en las sub-carpetas de /src, generé un archivo index.ts por cada una de ellas.

Finalmente, creé una carpeta assets al mismo nivel que la carpeta app, donde se almacenan todas las imágenes utilizadas en el proyecto.


Diseño de Componentes:

Este primer incremento se enfoca en el desarrollo de dos vistas principales: welcome y cv. Para su implementación, fue necesario crear algunos componentes adicionales, entre los cuales se encuentran toolbar y socialMedia. Estos componentes se cargarán siempre en la vista, independientemente de la redirección, ya que desde el componente principal (app.component) he configurado el contenido de cada vista para que incluya un toolbar, la ruta específica y el social media.

  • Toolbar:

    El componente "mat-toolbar" configura una barra de herramientas que he personalizado, incluye un botón de menú y un botón de usuario. El botón de menú, al hacer clic, despliega un menú donde se encuentran diferentes opciones de navegación, como "Home", "CV", "Project" y "Contact".

    Cada opción está representada por un icono y un texto descriptivo, facilitando la navegación entre las distintas vistas de la aplicación. Además, se ha integrado un segundo menú que permite acceder a las redes sociales. La barra de herramientas también incluye un espaciador que alinea adecuadamente los elementos, creando una interfaz de usuario ordenada y funcional.


  • Social-Media:

    El componente "app-social-media" se encarga de mostrar los enlaces a mis redes sociales disponibles en la aplicación. Utiliza un arreglo de objetos que representan cada red social, he incluido su nombre, enlace y ruta de imagen correspondiente. Cada enlace se presenta mediante el componente "app-social-media-element", que muestra un icono de la red social, al hacer clic redirige al usuario a la página correspondiente en una nueva pestaña.

    En el componente principal he incluido un contenedor que organiza visualmente los elementos de redes sociales y utiliza la directiva wobble para añadir un efecto de animación, mejorando así la interactividad y la experiencia del usuario.


  • Welcome:

    Este componente representa la vista por defecto de la web, donde se exhibe el nombre del proyecto junto con una presentación sobre el sitio. Además, incorpora un efecto fadeIn que se activa al cargar el componente, proporcionando una transición suave y atractiva para el usuario.


  • CV:

    En esta vista, se puede consultar mi currículum completo, organizado en secciones. Para acceder a cada una de estas secciones, he desarrollado un conjunto de botones que, al pulsarse, muestran el contenido correspondiente. Para realzar esta funcionalidad, he utilizado la directiva *ngIf, de manera que el contenido solo se mostrará si se cumple la condición establecida.

    Además, si se pulsa dos veces el mismo botón, se mostrará el contenido por defecto, es decir, la pestaña "Main".


Segundo Incremento:


Full-Stack Web:

En este incremento, el objetivo principal es transformar la aplicación front-end en una aplicación Full-Stack, integrando la parte back-end con Node.js. Tras iniciar el proyecto, decidí reestructurar los ficheros para mejorar la organización de la API y facilitar su escalabilidad.

Para ello, he creado un directorio principal llamado "src", que incluye subcarpetas de "config", "controllers", "middlewares", "models" y "routes". Dentro de "routes", se han definido las rutas que se enlazarán con los controladores y serán llamadas desde la aplicación Angular.


Base de datos MySQL:

Uno de los principales objetivos al desarrollar una web Full-Stack es poder comunicarme con una base de datos a través de los controladores en la parte back-end. Esto permite realizar consultas, agregar nueva información e implementar sesiones. Para gestionar la base de datos, he utilizado MySQL y he creado una única tabla que almacena los nombres, correos electrónicos y contraseñas cifradas de los usuarios registrados en la web.


Componentes y servicios de Usuarios:

Volviendo al front-end, al gestionar sesiones es necesario crear las vistas correspondientes, como register y login, que permiten registrarse e iniciar sesión respectivamente. Cada una de estas vistas incluye un formulario donde los usuarios pueden ingresar sus datos para completar el registro o autenticarse en la plataforma.

Para el registro, el componente register.component contiene la lógica necesaria para verificar que el formulario esté completamente rellenado y que los campos sean válidos. Por ejemplo, comprueba que las contraseñas introducidas coincidan. Una vez validada esta información, el componente llama a los métodos correspondientes de Usuario.service. Estos métodos se comunican con el backend, donde los controladores aplican la lógica pertinente. En este caso, se verifica que no exista una cuenta asociada al correo especificado y, de no ser así, otro controlador crea la cuenta.

Por otro lado, el componente de registro llamará a UsuarioService, que se encargará de comunicarse con la API para verificar si el correo y la contraseña proporcionados son correctos en comparación con los datos almacenados en la base de datos. Si la verificación es exitosa, la API enviará un token al Front-end para mantener activa la sesión. Para comprobar la sesión, he creado una nueva vista accesible desde el menú de usuario, llamada userInfo, que muestra el nombre y el correo del usuario en caso de que esté logueado.
